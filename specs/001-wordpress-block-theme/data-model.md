# Data Model: RenalInfoLK Web WordPress Block Theme

**Feature Branch**: `001-wordpress-block-theme`
**Date**: 2025-11-11
**Status**: Design Complete

## Overview

The RenalInfoLK Web WordPress block theme uses WordPress's native data structures exclusively. As a pure presentation layer theme, it does not define custom post types, taxonomies, or database tables. All content is managed through standard WordPress post types with semantic organization via categories and tags.

---

## 1. Post Types

### 1.1 Post (Standard Blog Posts)

**Purpose**: Medical articles, research updates, educational content, news announcements

**Post Type Identifier**: `post` (WordPress core)

**Fields (WordPress Standard)**:

| Field | Type | Description | Required |
|-------|------|-------------|----------|
| `post_title` | String (255) | Article headline | Yes |
| `post_content` | Long text | Full article content (HTML via block editor) | Yes |
| `post_excerpt` | Text | Manual excerpt or auto-generated summary | No |
| `post_status` | Enum | publish, draft, pending, private | Yes |
| `post_author` | User ID | Author (links to wp_users) | Yes |
| `post_date` | Datetime | Publication date and time | Yes |
| `post_modified` | Datetime | Last update timestamp | Auto |
| `post_name` | String | URL slug (e.g., kidney-disease-children) | Auto |
| `post_type` | String | Always "post" | Auto |
| `comment_status` | Enum | open, closed | Yes |
| `ping_status` | Enum | open, closed | Yes |

**Featured Image**: Attached media via `_thumbnail_id` post meta (recommended 1200x675px for 16:9 aspect ratio)

**Taxonomies**: Categories, Tags (see section 2)

**Usage in Theme**:
- Displayed in homepage featured posts grid (`templates/home.html`)
- Individual post view via `templates/single.html`
- Archive listings via `templates/archive.html` with responsive CSS Grid
- Search results via `templates/search.html`

**Example Medical Topics**:
- "Understanding Chronic Kidney Disease in Children"
- "Latest Research in Pediatric Nephrology"
- "Treatment Options for Nephrotic Syndrome"
- "Parent Guide: Managing Dialysis at Home"

### 1.3 Attachment (Media Library)

**Purpose**: Images, PDFs, documents referenced in posts and pages

**Post Type Identifier**: `attachment` (WordPress core)

**Fields (WordPress Standard)**:

| Field | Type | Description |
|-------|------|-------------|
| `post_title` | String | Media title |
| `post_content` | Text | Media description |
| `post_excerpt` | Text | Media caption |
| `post_mime_type` | String | MIME type (image/jpeg, image/webp, application/pdf) |
| `guid` | String | File URL |
| `post_parent` | Post/Page ID | Associated post (0 = unattached) |

**Post Meta**:
- `_wp_attached_file`: Relative path in uploads directory
- `_wp_attachment_metadata`: Array with dimensions, file size, subsizes (thumbnails)
- `_wp_attachment_image_alt`: Alt text for accessibility (critical for WCAG compliance)

**Image Sizes Generated by WordPress**:
- `thumbnail` (150x150, cropped)
- `medium` (300x300, max width/height)
- `medium_large` (768x0, max width)
- `large` (1024x1024, max width/height)
- `full` (original upload)

**Recommended Custom Image Sizes** (define in functions.php):
```php
add_image_size('archive-thumbnail', 400, 300, true); // 4:3 crop for blog grids
add_image_size('hero-image', 1200, 675, true); // 16:9 crop for hero sections
add_image_size('card-image', 600, 400, true); // 3:2 crop for content cards
```

**Usage in Theme**:
- Featured images in post archives and single posts
- Hero section backgrounds and illustrations
- Content section images in block patterns
- Medical diagrams, infographics, team photos

**Optimization Requirements**:
- Primary format: WebP with PNG/JPEG fallback
- Max file size: 500KB (recommend 200KB for web delivery)
- Alt text mandatory for accessibility
- Lazy loading enabled for below-fold images

## 2. Taxonomies

### 2.1 Category (Medical Topic Organization)

**Purpose**: Hierarchical organization of medical content by topic area

**Taxonomy Identifier**: `category` (WordPress core)

**Fields**:

| Field | Type | Description |
|-------|------|-------------|
| `term_id` | Integer | Unique category ID |
| `name` | String (200) | Category name (e.g., "Kidney Conditions") |
| `slug` | String (200) | URL-friendly identifier (e.g., kidney-conditions) |
| `term_group` | Integer | Grouping (unused in modern WP) |
| `parent` | Term ID | Parent category for hierarchy (0 = top-level) |
| `description` | Text | Category description (displayed in archives) |
| `count` | Integer | Number of posts in category |

**Recommended Category Structure for Medical Platform**:

```
Kidney Conditions (Parent)
├── Acute Kidney Injury
├── Chronic Kidney Disease
├── Nephrotic Syndrome
├── Glomerulonephritis
└── Polycystic Kidney Disease

Treatment Options (Parent)
├── Dialysis
├── Kidney Transplantation
├── Medication Management
└── Dietary Interventions

Patient Resources (Parent)
├── Understanding Diagnosis
├── Living with Kidney Disease
├── Family Support
└── Financial Assistance

Professional Guidelines (Parent)
├── Clinical Protocols
├── Research Publications
├── Continuing Education
└── Best Practices

News & Updates (Parent)
├── Research News
├── Event Announcements
├── Community Updates
└── Webinar Series
```

**Usage in Theme**:
- Archive pages filtered by category (`archive.html` template)
- Category display in post meta (single and archive views)
- Navigation menu integration (category links in Primary Menu)
- Pattern: `hidden-post-meta.php` displays categories with proper styling

**Archive URL Structure**:
- Single category: `/category/kidney-conditions/`
- Child category: `/category/kidney-conditions/chronic-kidney-disease/`

**SEO Considerations**:
- Category descriptions render in archive page headers
- Category pages indexed by search engines
- Breadcrumbs use category hierarchy

## 4. User Roles & Capabilities

### 4.1 WordPress Standard Roles

The theme does not modify WordPress user roles. Standard capabilities apply:

**Administrator**:
- Full site control (content, settings, theme, plugins, users)
- Can edit theme via Site Editor (Appearance → Editor)
- Can customize colors, typography, spacing via theme.json interface

**Editor**:
- Manage all posts, pages, categories, tags
- Cannot modify theme settings or install plugins
- Can use Site Editor to edit specific page layouts

**Author**:
- Create and publish own posts
- Upload images to Media Library
- Cannot edit others' posts or create pages

**Contributor**:
- Create posts but cannot publish (requires Editor/Admin approval)
- Cannot upload media files

**Subscriber**:
- Read-only access to content
- Can comment on posts (if comments enabled)

### 4.2 Content Creation Workflow

**Medical Content Editors (Editor Role)**:
1. Create new post via Posts → Add New
2. Add title, write content using block editor
3. Assign categories (e.g., "Kidney Conditions")
4. Add tags (e.g., "pediatric-nephrology", "for-parents")
5. Set featured image (recommended: 1200x675px, WebP format)
6. Write excerpt (150-200 characters for archive listings)
7. Publish or schedule for future publication

**Page Administrators (Administrator Role)**:
1. Create new page via Pages → Add New
2. Insert block patterns from pattern inserter (heroes, CTAs, content sections)
3. Customize pattern content inline (text, images, buttons)
4. Set page hierarchy (parent/child relationship)
5. Publish page

**Navigation Menu Management (Administrator Role)**:
1. Appearance → Menus
2. Create new menu or edit existing
3. Add pages, posts, categories, custom links
4. Organize items (drag-and-drop hierarchy)
5. Assign menu to location (Primary, Footer)

## 5. Media Management

### 5.1 Image Handling

**Upload Process**:
1. Media → Add New → Upload files
2. WordPress auto-generates responsive image sizes
3. Add alt text (mandatory for accessibility)
4. Optional: Add caption and description

**Recommended Image Specifications**:

| Usage | Dimensions | Aspect Ratio | Format | Max File Size |
|-------|------------|--------------|--------|---------------|
| Featured Image | 1200x675 | 16:9 | WebP | 200KB |
| Hero Background | 1920x1080 | 16:9 | WebP | 300KB |
| Content Card | 600x400 | 3:2 | WebP | 100KB |
| Logo | 300x100 | 3:1 | SVG or PNG | 20KB |
| Team Photo | 400x400 | 1:1 | WebP | 80KB |

**Image Optimization**:
- Primary: WebP format (supported by 95%+ browsers)
- Fallback: JPEG for photos, PNG for graphics
- Compression: 80-85% quality for JPEG/WebP
- Tools: ImageOptim, TinyPNG, Squoosh.app

**Lazy Loading**:
- WordPress automatically applies `loading="lazy"` to images below fold
- Above-fold hero images use `loading="eager"` for immediate display

**Responsive Images**:
- WordPress generates `srcset` attribute with multiple sizes
- Browser selects appropriate size based on viewport
- Example: `srcset="image-400.webp 400w, image-800.webp 800w, image-1200.webp 1200w"`

### 5.2 Document Handling

**Supported Document Types**:
- PDF (recommended for medical protocols, research papers)
- DOCX (editable documents for download)
- PPT/PPTX (presentation slides)

**Upload Restrictions** (WordPress defaults):
- Max file size: 64MB (server-dependent, configurable in wp-config.php)
- Allowed MIME types: PDF, DOCX, PPTX, JPG, PNG, WebP, GIF, SVG

**Document Usage**:
- Link to PDFs from posts/pages: `<!-- wp:file /-->` block
- Display PDF thumbnails with preview link
- Provide file size and type information for user transparency

## 6. Search Functionality

### 6.1 WordPress Native Search

**Search Scope**:
- Post titles and content
- Page titles and content
- Excerpts (if present)

**Excluded from Search** (by default):
- Draft posts
- Private posts
- Trashed content
- Attachments (Media Library files)

**Search Implementation**:
1. User enters query in header search form
2. Form submits to `?s={query}` URL parameter
3. WordPress loads `templates/search.html` template
4. Results display via `<!-- wp:query {"query":{"postType":"post","search":true}} /-->`

**Search Results Display** (`templates/search.html`):
- Query term highlighted: "Results for: [query]"
- Result count: "Found X results"
- Each result shows: title, excerpt, date, featured image
- Pagination for 10+ results
- Zero results: "No results found" message with search tips

### 6.2 Search Enhancement (Optional - via Plugins)

**Recommended Search Plugins** (not included in theme, user-installable):
- **Relevanssi**: Better relevance ranking, fuzzy matching
- **SearchWP**: Custom field search, taxonomy search
- **Jetpack Search**: Instant search with filters (hosted by WordPress.com)

**Advanced Search Features** (if plugins installed):
- Filter by category/tag
- Sort by relevance, date, popularity
- Live search (AJAX results as you type)
- Synonym mapping (e.g., "CKD" → "chronic kidney disease")

## 7. Comments System

### 7.1 WordPress Native Comments

**Comment Data Structure**:

| Field | Type | Description |
|-------|------|-------------|
| `comment_ID` | Integer | Unique comment ID |
| `comment_post_ID` | Post ID | Associated post |
| `comment_author` | String | Commenter name |
| `comment_author_email` | Email | Commenter email (not public) |
| `comment_content` | Text | Comment text |
| `comment_date` | Datetime | Comment submission timestamp |
| `comment_approved` | Boolean | 0 = pending, 1 = approved, spam = spam |
| `comment_parent` | Comment ID | Parent comment for threaded replies |

**Comment Moderation**:
- Admin approval required: Settings → Discussion → "Comment must be manually approved"
- Spam filtering: Akismet plugin (recommended)
- Threaded replies: Up to 5 levels deep

**Usage in Theme**:
- Comments section in `templates/single.html` via `<!-- wp:post-comments /-->` block
- Comment form via `<!-- wp:post-comments-form /-->` block
- Display: Avatar (Gravatar), author name, date, comment text, reply link

**Recommendation for Medical Platform**:
- **Disable comments** on medical content to avoid misinformation
- Alternative: Use contact form for questions (directs to healthcare professionals)
- If comments enabled: Require manual approval, monitor closely for accuracy

## 9. Internationalization (i18n)

### 9.1 Translation Readiness

**Text Domain**: `renalinfo-web` (must match theme slug)

**Translatable Strings**: All theme-generated text wrapped in translation functions:
```php
__('Hello World', 'renalinfo-web')          // Simple translation
_e('Hello World', 'renalinfo-web')          // Echo translation
esc_html__('Hello', 'renalinfo-web')        // Escaped translation
```

**Translation Files Location**: `/languages/`
- `renalinfo-web.pot`: Translation template (generated by developers)
- `renalinfo-web-es_ES.po/.mo`: Spanish translation (if available)
- `renalinfo-web-fr_FR.po/.mo`: French translation

**WordPress Translation Integration**:
- Translations managed via WordPress.org translation portal (if theme submitted)
- Users install language packs via Settings → General → Site Language
- Theme automatically loads correct language file

### 9.2 Content Translation (Multilingual Sites)

**Theme Does Not Include Multilingual Functionality**

**Recommended Multilingual Plugins** (user-installable):
- **WPML** (premium, comprehensive)
- **Polylang** (free, user-friendly)
- **TranslatePress** (free, visual translation)

**How Multilingual Plugins Work**:
1. Duplicate posts/pages for each language
2. Switch language via menu or widget
3. Translate categories, tags, menus separately
4. Each language has unique URL structure (e.g., `/en/about/`, `/es/acerca-de/`)

**Theme Compatibility**: RenalInfoLK Web theme is compatible with all major multilingual plugins (no custom post type conflicts).

## 11. Theme Data Model Summary

**Key Principles**:
1. **No Custom Post Types**: Theme uses WordPress core post types only (Post, Page, Attachment)
2. **No Custom Taxonomies**: Theme uses WordPress core taxonomies only (Category, Tag)
3. **No Custom Database Tables**: All data stored in standard WordPress tables
4. **Plugin Flexibility**: Medical platform can add custom functionality via plugins without theme dependency
5. **Future-Proof**: Theme remains compatible with WordPress updates and plugin ecosystem

**Medical Content Strategy**:
- **Posts**: Timely content (news, research updates, blog articles)
- **Pages**: Evergreen content (about, services, resources, contact)
- **Categories**: Medical topic organization (hierarchical)
- **Tags**: Cross-referencing and discovery (non-hierarchical)
- **Navigation Menus**: Site structure and user wayfinding
- **Media Library**: Images, documents, medical resources

**Data Ownership**:
- All content remains in WordPress database (portable)
- Theme can be changed without losing content
- Export via WordPress Tools → Export (XML format)
- Import via WordPress Tools → Import
